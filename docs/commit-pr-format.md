# Commit And PR Format

This repository enforces conventional commit messages and structured pull requests.

## Commit format (mandatory)

Use:

```text
type(scope): summary
```

Allowed `type` values:

- `feat`
- `fix`
- `docs`
- `style`
- `refactor`
- `perf`
- `test`
- `build`
- `ci`
- `chore`
- `revert`

Rules:

- `summary` must be specific and action-oriented.
- Keep the first line concise.
- Do not add AI attribution or co-author metadata.

Examples:

- `feat(catalog): add dynamic pricing rules`
- `fix(auth): prevent token refresh loop`
- `chore(release): deploy catalog to production`

## Pull request format (mandatory)

PR title must follow the same conventional commit format.

PR body should use `.github/pull_request_template.md` sections:

- Summary
- Scope
- Changes
- Verification
- Security impact
- Rollback plan
- Deployment notes
- Checklist

## Release pipeline behavior

`scripts/release-pipeline.sh` validates:

- commit message format before commit
- PR title format before PR creation

If no PR body is provided, it uses:

1. `--pr-body-file` (if provided)
2. `.github/pull_request_template.md` (if present)
3. fallback autogenerated text
